## 核心业务表

### 1. users - 用户表

存储系统用户信息，包括普通用户、管理员和超级管理员。

#### 表结构

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | int | 主键，自增 |
| `username` | varchar(20) | 用户名，唯一索引 |
| `password` | varchar(255) | 密码（哈希存储） |
| `display_name` | varchar(20) | 显示名称 |
| `role` | int | 角色：0=访客, 1=普通用户, 10=管理员, 100=超级管理员 |
| `status` | int | 状态：1=启用, 2=禁用 |
| `email` | varchar(50) | 邮箱，索引 |
| `github_id` | varchar(64) | GitHub OAuth ID |
| `discord_id` | varchar(64) | Discord OAuth ID |
| `oidc_id` | varchar(64) | OIDC ID |
| `wechat_id` | varchar(64) | 微信 ID |
| `telegram_id` | varchar(64) | Telegram ID |
| `linux_do_id` | varchar(64) | LinuxDO OAuth ID |
| `access_token` | varchar(32) | 系统管理令牌 |
| `quota` | int | 当前配额 |
| `used_quota` | int | 已使用配额 |
| `request_count` | int | 请求次数 |
| `group` | varchar(64) | 用户分组，默认 'default' |
| `aff_code` | varchar(32) | 邀请码，唯一索引 |
| `aff_count` | int | 邀请人数 |
| `aff_quota` | int | 邀请剩余额度 |
| `aff_history_quota` | int | 邀请历史总额度 |
| `inviter_id` | int | 邀请人 ID |
| `setting` | text | 用户设置（JSON） |
| `remark` | varchar(255) | 备注 |
| `stripe_customer` | varchar(64) | Stripe 客户 ID |
| `created_at` | bigint | 创建时间（Unix 时间戳） |
| `last_login_at` | bigint | 最后登录时间 |
| `deleted_at` | timestamp | 软删除时间 |

#### API 接口

| 方法 | 路径 | 权限 | 功能 |
|------|------|------|------|
| POST | `/api/user/register` | 公开 | 用户注册 |
| POST | `/api/user/login` | 公开 | 用户登录 |
| POST | `/api/user/login/2fa` | 公开 | 2FA 验证登录 |
| GET | `/api/user/self` | 用户 | 获取当前用户信息 |
| PUT | `/api/user/self` | 用户 | 更新当前用户信息 |
| DELETE | `/api/user/self` | 用户 | 删除当前用户 |
| GET | `/api/user/` | 管理员 | 获取所有用户列表 |
| GET | `/api/user/search` | 管理员 | 搜索用户 |
| GET | `/api/user/:id` | 管理员 | 获取指定用户 |
| POST | `/api/user/` | 管理员 | 创建用户 |
| PUT | `/api/user/` | 管理员 | 更新用户 |
| DELETE | `/api/user/:id` | 管理员 | 删除用户 |
| POST | `/api/user/manage` | 管理员 | 管理用户（批量操作） |

#### 核心功能

- **用户注册/登录**：支持密码登录、OAuth 登录（GitHub、Discord、OIDC、微信、Telegram、LinuxDO）
- **权限管理**：基于角色的访问控制（RBAC）
- **配额管理**：用户配额的增加、扣减、查询
- **邀请系统**：用户邀请码生成、邀请奖励

### 2. channels - 渠道表

存储上游 AI 提供商配置，是 API 转发的核心。

#### 表结构

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | int | 主键，自增 |
| `type` | int | 渠道类型（1=OpenAI, 2=API2D, 3=Azure, 等） |
| `key` | text | API Key（支持多 Key） |
| `openai_organization` | varchar(64) | OpenAI 组织 ID |
| `test_model` | varchar(64) | 测试模型 |
| `status` | int | 状态：1=启用, 2=手动禁用, 3=自动禁用 |
| `name` | varchar(255) | 渠道名称 |
| `weight` | uint | 权重（负载均衡） |
| `created_time` | bigint | 创建时间 |
| `test_time` | bigint | 最后测试时间 |
| `response_time` | int | 响应时间（毫秒） |
| `base_url` | varchar(255) | 自定义 Base URL |
| `other` | text | 其他信息 |
| `balance` | float | 余额（美元） |
| `balance_updated_time` | bigint | 余额更新时间 |
| `models` | text | 支持的模型列表（逗号分隔） |
| `group` | varchar(64) | 分组，默认 'default' |
| `used_quota` | bigint | 已使用配额 |
| `model_mapping` | text | 模型映射（JSON） |
| `status_code_mapping` | varchar(1024) | 状态码映射 |
| `priority` | bigint | 优先级（越大越优先） |
| `auto_ban` | int | 是否自动禁用（1=是） |
| `other_info` | text | 其他信息（JSON） |
| `tag` | varchar(64) | 标签 |
| `setting` | text | 渠道设置（JSON） |
| `param_override` | text | 参数覆盖（JSON） |
| `header_override` | text | 请求头覆盖（JSON） |
| `remark` | varchar(255) | 备注 |
| `channel_info` | json | 多 Key 模式信息 |
| `settings` | text | 其他设置（如 Azure 版本） |

#### 渠道类型对照表

| 值 | 类型 | 说明 |
|----|------|------|
| 1 | OpenAI | OpenAI 官方 API |
| 2 | API2D | API2D 代理 |
| 3 | Azure | Azure OpenAI |
| 4 | CloseAI | CloseAI |
| 8 | Claude | Anthropic Claude |
| 9 | ClaudeProxy | Claude 代理 |
| 10 | Palm | Google PaLM |
| 11 | MaaS | MaaS 平台 |
| 14 | Gemini | Google Gemini |
| 15 | Zhipu | 智谱 AI |
| 16 | Ollama | Ollama 本地模型 |
| 17 | DeepSeek | DeepSeek |
| 18 | Moonshot | Moonshot AI |
| ... | ... | 更多类型见 `constant/channel.go` |

#### API 接口

| 方法 | 路径 | 权限 | 功能 |
|------|------|------|------|
| GET | `/api/channel/` | 管理员 | 获取所有渠道 |
| GET | `/api/channel/search` | 管理员 | 搜索渠道 |
| GET | `/api/channel/:id` | 管理员 | 获取指定渠道 |
| POST | `/api/channel/:id/key` | 超级管理员 | 获取渠道 Key（需二次验证） |
| POST | `/api/channel/` | 管理员 | 创建渠道 |
| PUT | `/api/channel/` | 管理员 | 更新渠道 |
| DELETE | `/api/channel/:id` | 管理员 | 删除渠道 |
| POST | `/api/channel/batch` | 管理员 | 批量删除渠道 |
| GET | `/api/channel/test` | 管理员 | 测试所有渠道 |
| GET | `/api/channel/test/:id` | 管理员 | 测试指定渠道 |
| GET | `/api/channel/update_balance` | 管理员 | 更新所有渠道余额 |
| POST | `/api/channel/tag/disabled` | 管理员 | 按标签禁用渠道 |
| POST | `/api/channel/tag/enabled` | 管理员 | 按标签启用渠道 |
| GET | `/api/channel/fetch_models/:id` | 管理员 | 获取渠道支持的模型 |

#### 核心功能

- **多 Key 支持**：单个渠道支持多个 API Key，支持轮询和随机模式
- **负载均衡**：基于优先级和权重的负载均衡
- **模型映射**：将请求模型映射到实际模型
- **自动禁用**：渠道异常时自动禁用
- **标签管理**：按标签批量管理渠道

### 3. tokens - 令牌表

存储用户的 API Token（API Key），用于 API 调用认证。

#### 表结构

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | int | 主键，自增 |
| `user_id` | int | 所属用户 ID，索引 |
| `key` | varchar(128) | Token 值，唯一索引 |
| `status` | int | 状态：1=启用, 2=禁用, 3=过期, 4=耗尽 |
| `name` | varchar(255) | Token 名称 |
| `created_time` | bigint | 创建时间 |
| `accessed_time` | bigint | 最后访问时间 |
| `expired_time` | bigint | 过期时间（-1=永不过期） |
| `remain_quota` | int | 剩余配额 |
| `unlimited_quota` | bool | 是否无限配额 |
| `model_limits_enabled` | bool | 是否启用模型限制 |
| `model_limits` | text | 允许的模型列表（逗号分隔） |
| `allow_ips` | text | 允许的 IP 列表 |
| `used_quota` | int | 已使用配额 |
| `group` | varchar(64) | 分组 |
| `cross_group_retry` | bool | 是否允许跨分组重试 |
| `deleted_at` | timestamp | 软删除时间 |

#### API 接口

| 方法 | 路径 | 权限 | 功能 |
|------|------|------|------|
| GET | `/api/token/` | 用户 | 获取用户所有 Token |
| GET | `/api/token/search` | 用户 | 搜索 Token |
| GET | `/api/token/:id` | 用户 | 获取指定 Token |
| POST | `/api/token/:id/key` | 用户 | 获取 Token 明文值 |
| POST | `/api/token/` | 用户 | 创建 Token |
| PUT | `/api/token/` | 用户 | 更新 Token |
| DELETE | `/api/token/:id` | 用户 | 删除 Token |
| POST | `/api/token/batch` | 用户 | 批量删除 Token |

#### 核心功能

- **Token 生成**：生成 `sk-` 前缀的 API Key
- **配额控制**：支持无限配额和固定配额
- **模型限制**：限制 Token 可访问的模型
- **IP 限制**：限制 Token 可使用的 IP
- **过期控制**：支持设置过期时间

## 配置与缓存

### Redis 缓存使用

| 缓存键模式 | 说明 |
|-----------|------|
| `new-api:user:status:{id}` | 用户状态缓存 |
| `new-api:user:quota:{id}` | 用户配额缓存 |
| `new-api:user:group:{id}` | 用户分组缓存 |
| `new-api:token:{key}` | Token 缓存 |
| `new-api:channel:{id}` | 渠道缓存 |

### 内存缓存

- **渠道缓存**：`model/channel_cache.go` - 启用内存缓存时使用
- **用户缓存**：`model/user_cache.go` - 用户基本信息缓存
- **Token 缓存**：`model/token_cache.go` - Token 验证缓存
