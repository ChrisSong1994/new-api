## 项目概览

7X API 是一个 AI API 网关/代理服务，将 40+ 上游 AI 提供商（OpenAI、Claude、Gemini 等）聚合在统一的 API 后面，提供：

- 用户管理和认证（JWT、OAuth、WebAuthn）
- 计费和配额管理
- 速率限制
- 管理仪表板

### 技术栈

| 层级 | 技术 |
|------|------|
| 后端框架 | Go 1.22+ / Gin |
| ORM | GORM v2 |
| 前端 | React 19 / Rsbuild / Radix UI (default) 或 React 18 / Vite / Semi Design (classic) |
| 数据库 | SQLite / MySQL / PostgreSQL |
| 缓存 | Redis + 内存缓存 |

## 阅读顺序建议

### 初学者路线

1. **main.go** - 理解启动流程
2. **router/** - 了解路由注册
3. **middleware/auth.go** - 认证逻辑
4. **controller/relay.go** - 核心 API 转发入口
5. **relay/** - 各提供商适配器
6. **model/** - 数据模型
7. **service/billing.go** - 计费逻辑

### 按功能模块路线

| 功能模块 | 核心文件 | 阅读重点 |
|----------|----------|----------|
| API 转发 | `controller/relay.go` → `relay/` | 请求转发、响应流式处理 |
| 认证授权 | `middleware/auth.go` → `controller/auth.go` | JWT、OAuth、Session |
| 渠道管理 | `model/channel.go` → `controller/channel.go` | 渠道选择、负载均衡 |
| 计费系统 | `service/billing.go` → `service/quota.go` | 额度计算、扣费逻辑 |
| 用户管理 | `model/user.go` → `controller/user.go` | 用户注册、权限管理 |
| Token 管理 | `model/token.go` → `controller/token.go` | API Key 生成、验证 |

**认证中间件流程** (`middleware/auth.go`):

```
请求 → 提取 Token (Header/Query)
         │
         ▼
    Token 验证 (JWT/Session)
         │
         ├── 失败 → 返回 401
         │
         └── 成功 → 设置用户信息到 Context
                        │
                        ▼
                   权限检查
                        │
                        ├── 无权限 → 返回 403
                        │
                        └── 有权限 → 继续处理
```

**请求分发中间件** (`middleware/distributor.go`):

负责根据请求模型选择合适的渠道 (Channel)，核心逻辑：

1. 解析请求中的模型名称
2. 根据模型匹配可用渠道
3. 应用负载均衡策略
4. 将选中的渠道信息存入 Context

### 7. Relay 转发层

**目录**: `relay/`

这是项目最核心的部分，负责将请求转发到上游 AI 提供商。

```
relay/
├── relay.go              # 转发入口
├── adaptor.go            # 适配器接口
├── common/               # 通用处理逻辑
│   ├── relay.go
│   └── ...
├── helper/               # 辅助函数
├── channel/              # 各提供商适配器
│   ├── openai/           # OpenAI 适配器
│   ├── claude/           # Claude 适配器
│   ├── gemini/           # Gemini 适配器
│   ├── aws/              # AWS Bedrock 适配器
│   ├── azure/            # Azure OpenAI 适配器
│   └── ...               # 其他 40+ 提供商
├── constant/             # 常量定义
└── reasonmap/            # 错误原因映射
```

**适配器接口** (`relay/adaptor.go`):

```go
type Adaptor interface {
    Init(*dto.RelayInfo)           // 初始化
    GetRequestURL() string         // 获取请求 URL
    SetupRequestHeader()           // 设置请求头
    ConvertRequest() any           // 转换请求格式
    DoRequest() (*http.Response, error) // 发送请求
    DoResponse() (usage *dto.Usage, err *dto.Error) // 处理响应
}
```

**转发流程**:

```
Relay 请求
    │
    ▼
┌──────────────────┐
│ 解析请求模型     │
└──────────────────┘
    │
    ▼
┌──────────────────┐
│ 选择渠道         │ ← middleware/distributor.go
└──────────────────┘
    │
    ▼
┌──────────────────┐
│ 获取适配器       │ ← relay/adaptor.go
│ (根据渠道类型)   │
└──────────────────┘
    │
    ▼
┌──────────────────┐
│ 转换请求格式     │ ← OpenAI → Claude/Gemini/...
└──────────────────┘
    │
    ▼
┌──────────────────┐
│ 发送请求         │
└──────────────────┘
    │
    ▼
┌──────────────────┐
│ 处理响应         │
│ (流式/非流式)    │
└──────────────────┘
    │
    ▼
┌──────────────────┐
│ 计费 & 记录日志  │
└──────────────────┘
```

### 2. 渠道选择逻辑

**文件**: `model/channel_select.go`, `middleware/distributor.go`

```
渠道选择算法:
    │
    ├── 1. 根据模型过滤可用渠道
    │
    ├── 2. 根据用户组过滤
    │
    ├── 3. 按优先级排序
    │
    ├── 4. 同优先级内按权重负载均衡
    │
    └── 5. 返回选中的渠道
```

### 3. 计费流程

**文件**: `service/billing.go`, `service/quota.go`

```
计费流程:
    │
    ├── 预扣费 (PreConsumeQuota)
    │   - 在请求发送前预扣配额
    │   - 防止用户超额使用
    │
    ├── 请求处理
    │
    └── 结算 (PostConsumeQuota)
        - 根据实际使用量计算费用
        - 多退少补
```
