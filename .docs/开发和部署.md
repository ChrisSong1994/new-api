## 本地开发

### 方式一：Docker Compose 开发环境（推荐）

使用 Docker Compose 可以快速搭建完整的开发环境，包括后端服务、数据库和 Redis。

#### 1. 启动后端服务

```bash
# 启动开发环境（PostgreSQL + Redis + 后端）
docker compose -f docker-compose.dev.yml up -d

# 查看日志
docker compose -f docker-compose.dev.yml logs -f new-api
```

后端服务将在 `http://localhost:3000` 启动。

#### 2. 启动前端开发服务器

**Default 前端（React 19 + Rsbuild + Radix UI）：**

```bash
cd web/default

# 安装依赖
bun install

# 启动开发服务器
bun run dev
```

前端开发服务器将在 `http://localhost:3001` 启动，API 请求会自动代理到 `localhost:3000`。

#### 3. 修改代码后重新构建后端

当修改 Go 代码后，需要重新构建后端镜像：

```bash
docker compose -f docker-compose.dev.yml up -d --build new-api
```

#### 4. 停止开发环境

```bash
# 停止服务
docker compose -f docker-compose.dev.yml down

# 停止并删除数据卷（重置数据）
docker compose -f docker-compose.dev.yml down -v
```

### 方式二：本地直接运行

如果你希望在本地直接运行而不使用 Docker，请按以下步骤操作。

#### 1. 准备数据库

**使用 SQLite（最简单）：**

无需额外配置，项目会自动创建 SQLite 数据库。

**使用 PostgreSQL：**

```bash
# 创建数据库
createdb new-api
```

**使用 MySQL：**

```sql
CREATE DATABASE new-api CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

#### 2. 配置环境变量

复制环境变量示例文件：

```bash
cp .env.example .env
```

编辑 `.env` 文件，配置数据库连接：

```bash
# SQLite（默认，无需配置）
# 或 PostgreSQL
SQL_DSN=postgresql://user:password@localhost:5432/new-api

# 或 MySQL
SQL_DSN=user:password@tcp(127.0.0.1:3306)/new-api?parseTime=true

# Redis（可选）
REDIS_CONN_STRING=redis://localhost:6379
```

#### 3. 启动后端

```bash
# 安装 Go 依赖
go mod download

# 运行后端
go run main.go
```

后端将在 `http://localhost:3000` 启动。

#### 4. 启动前端

在另一个终端中：

```bash
cd web/default
bun install
bun run dev
```

## 生产部署

### Docker 部署（推荐）

#### 1. 使用预构建镜像

项目提供了预构建的 Docker 镜像：

```bash
# 使用 docker-compose.yml 部署
docker compose up -d
```

这会拉取 `calciumion/new-api:latest` 镜像并启动完整的服务栈。

#### 2. 自定义构建

如需自定义构建：

```bash
# 构建镜像
docker build -t new-api:custom .

# 运行容器
docker run -d \
  --name new-api \
  -p 3000:3000 \
  -v ./data:/data \
  -e SQL_DSN="postgresql://user:password@postgres:5432/new-api" \
  -e REDIS_CONN_STRING="redis://:password@redis:6379" \
  new-api:custom
```

#### 3. Docker Compose 生产配置

编辑 `docker-compose.yml`，修改以下配置：

```yaml
environment:
  # 修改数据库密码
  - SQL_DSN=postgresql://root:YOUR_STRONG_PASSWORD@postgres:5432/new-api
  # 修改 Redis 密码
  - REDIS_CONN_STRING=redis://:YOUR_STRONG_PASSWORD@redis:6379
  # 设置会话密钥
  - SESSION_SECRET=your_random_session_secret
  # 设置时区
  - TZ=Asia/Shanghai
```

**重要：生产环境请务必修改默认密码！**

### 手动构建部署

#### 1. 构建前端

```bash
cd web/default
bun install
bun run build
```

构建产物位于 `web/default/dist/`。

#### 2. 构建后端

```bash
# 回到项目根目录
cd ../..

# 构建后端（前端 dist 目录会被嵌入到二进制文件中）
go build -ldflags "-s -w" -o new-api
```

#### 3. 运行

```bash
# 设置环境变量
export SQL_DSN="postgresql://user:password@localhost:5432/new-api"
export REDIS_CONN_STRING="redis://localhost:6379"

# 运行
./new-api
```

## 配置说明

### 环境变量

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| `PORT` | 服务端口 | `3000` |
| `SQL_DSN` | 数据库连接字符串 | SQLite |
| `REDIS_CONN_STRING` | Redis 连接字符串 | - |
| `SESSION_SECRET` | 会话密钥（多节点部署必须设置） | - |
| `TZ` | 时区 | `Asia/Shanghai` |
| `BATCH_UPDATE_ENABLED` | 启用批量更新 | `true` |
| `ERROR_LOG_ENABLED` | 启用错误日志 | `true` |
| `STREAMING_TIMEOUT` | 流模式超时时间（秒） | `120` |

### 数据库支持

项目支持三种数据库：

1. **SQLite**（默认）：无需配置，适合开发和小规模部署
2. **PostgreSQL**（推荐生产使用）：高性能，支持大规模数据
3. **MySQL 5.7.8+**：兼容性好

### 前端主题

项目使用 React 19 + Rsbuild + Radix UI + Tailwind CSS 构建前端。

### 日常开发步骤

1. **启动后端**：`docker compose -f docker-compose.dev.yml up -d`
2. **启动前端**：`cd web/default && bun run dev`
3. **修改代码**：前端热更新自动生效，后端需重新构建
4. **测试验证**：访问 `http://localhost:3001`
5. **提交代码**：确保测试通过后提交

### 常用命令速查

```bash
# === Docker 开发环境 ===
docker compose -f docker-compose.dev.yml up -d      # 启动开发环境
docker compose -f docker-compose.dev.yml logs -f    # 查看日志
docker compose -f docker-compose.dev.yml down       # 停止环境
docker compose -f docker-compose.dev.yml down -v    # 重置数据

# === 前端开发 ===
cd web/default && bun install                        # 安装依赖
bun run dev                                          # 启动开发服务器
bun run build                                        # 构建生产版本
bun run i18n:sync                                    # 同步国际化文件

# === 后端开发 ===
go mod download                                      # 下载依赖
go run main.go                                       # 运行开发服务器
go build -o new-api                                  # 构建二进制

# === 生产部署 ===
docker compose up -d                                 # 启动生产环境
docker compose logs -f                               # 查看日志
docker compose down                                  # 停止服务
```

## 故障排除

### 常见问题

#### Q: 前端无法连接后端 API

确保后端服务正在运行，检查前端开发服务器的代理配置。前端代理配置在 `web/default/rsbuild.config.ts`。

#### Q: 数据库连接失败

检查 `SQL_DSN` 环境变量配置是否正确，确保数据库服务已启动。

#### Q: Redis 连接失败

如果未配置 Redis，项目会使用内存缓存。如需 Redis，确保 `REDIS_CONN_STRING` 配置正确。

#### Q: Docker 构建失败

确保 Docker 有足够的内存（建议 4GB+），并检查网络连接是否正常。
